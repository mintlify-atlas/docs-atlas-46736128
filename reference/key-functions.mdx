**Purpose:** Walks a raw PE image in a heap buffer to find the first resource data entry in the

<Accordion title="RaiseExceptionInThread()">

**File:** `FunnyApp.cpp:251`

```cpp
void RaiseExceptionInThread(HANDLE hthread)
```

**Purpose:** Redirects the instruction pointer of a suspended thread to `ThrowFunc` so that a
C++ exception is raised in that thread's context. Used by `CallWD` to abort `wmain` when an RPC
setup error occurs.

**Parameters:**
- `hthread` — handle to the thread to redirect (must have `THREAD_SUSPEND_RESUME |
  THREAD_GET_CONTEXT | THREAD_SET_CONTEXT` access).

**Return value:** None.

**Key implementation:**

```cpp
void ThrowFunc() { throw 0; }

void RaiseExceptionInThread(HANDLE hthread)
{
    CONTEXT ctx = { 0 };
    ctx.ContextFlags = CONTEXT_FULL;
    SuspendThread(hthread);
    if (GetThreadContext(hthread, &ctx))
    {
        ctx.Rip = (DWORD64)ThrowFunc;
        SetThreadContext(hthread, &ctx);
        ResumeThread(hthread);
    }
}
```

This technique only works reliably on x64 because `Rip` is a 64-bit register. The approach is
fragile: if the target thread holds a lock or is in a system call at the time of suspension, the
redirected execution will behave unpredictably.

</Accordion>
