BlueHammer is a proof-of-concept exploit that chains several Windows primitives together to read files that are normally inaccessible — including the live SAM database — and subsequently escalate privileges by decrypting local account NTLM hashes.

<Warning>
  This repository is published for **security research and educational purposes only**. The code demonstrates a real vulnerability chain. Running it on systems you do not own or have explicit permission to test is illegal and unethical.
</Warning>

## Attack overview

The exploit chains four distinct stages:

<Steps>
  <Step title="Trigger a Windows Defender signature update">
    BlueHammer waits for a pending WD signature update, downloads the update package directly, and calls `ServerMpUpdateEngineSignature` over ALPC/RPC to instruct WD to process it from an attacker-controlled directory.
  </Step>
  <Step title="Create and freeze a Volume Shadow Copy">
    An EICAR test file causes WD to perform a scan. During the scan, a Cloud Files sync-root callback intercepts WD's directory query and uses an oplock on a `.lock` file to freeze WD's I/O at a precise moment — while a Volume Shadow Copy of the system volume is created.
  </Step>
  <Step title="Redirect WD's file read to the SAM hive">
    With WD frozen and the VSS mounted, a junction + NT object-manager symbolic link redirects WD's next privileged file open (`mpasbase.vdm`) to `\Device\HarddiskVolumeShadowCopyN\Windows\System32\Config\SAM`. WD reads the SAM file on behalf of the attacker.
  </Step>
  <Step title="Decrypt hashes and escalate privileges">
    The leaked SAM hive is parsed offline. Using the LSA boot key extracted from the live registry, all local account NTLM hashes are decrypted. Passwords are temporarily changed to log on as each admin user, launch a SYSTEM-level service, and then restored.
  </Step>
</Steps>

## Key techniques

<CardGroup cols={2}>
  <Card title="WD RPC abuse" icon="network-wired" href="/vulnerability/windows-defender-update-abuse">
    Calling `ServerMpUpdateEngineSignature` (Proc42) over the Windows Defender ALPC port with an attacker-controlled directory path.
  </Card>
  <Card title="Oplock + Cloud Files freeze" icon="lock" href="/vulnerability/vss-freeze-technique">
    Using `FSCTL_REQUEST_BATCH_OPLOCK` and the Cloud Files API to freeze Windows Defender at a deterministic I/O point.
  </Card>
  <Card title="NT symlink injection" icon="link" href="/vulnerability/sam-extraction">
    Creating NT object-manager symbolic links under `\BaseNamedObjects\Restricted` to redirect WD's privileged file read.
  </Card>
  <Card title="Offline SAM decryption" icon="key" href="/reference/key-functions">
    Parsing the SAM hive with `offreg.dll`, extracting the boot key, and decrypting NTLM hashes with AES-128-CBC + DES-ECB.
  </Card>
</CardGroup>
